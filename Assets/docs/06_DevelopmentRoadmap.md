# 概要

本ドキュメントは、「もじぺっと」の開発順序を定義する。

目的は、MVP（Minimum Viable Product）を最短で完成させることである。

追加機能はMVP完成後に実装する。

---

# 開発方針

優先順位は以下とする。

1. 動作すること
2. セーブできること
3. ゲームとして遊べること
4. 演出を追加すること
5. 最適化すること

アニメーションや演出は最後に実装する。

# MVP完成条件

以下を満たすこと。

・ゲーム開始できる

・文字を取得できる

・文字を育成できる

・単語を登録できる

・研究できる

・放置報酬を受け取れる

・ショップが利用できる

・図鑑が動作する

・施設を強化できる

・セーブ・ロードできる

ロードが高速。

60FPSを維持する。

ロード時間5秒以内。

バッテリー消費を抑える。

# 完成基準

ゲームデザイン仕様書で定義した機能がすべて実装されていること。

重大なバグが存在しないこと。

セーブデータ互換性を維持していること。

ゲームループが継続して成立していること。
